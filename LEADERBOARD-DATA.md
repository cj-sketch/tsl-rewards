# Лидерборд tsl-skills.com — как менять данные

Все данные сайта (две доски + каталог карточек) лежат в **одном файле — `data.json`** в корне этого репо.
Сайт грузит его в рантайме. Меняешь цифры/карточки → правишь `data.json` → деплоишь. Сборка фронта (node/npm) **не нужна** — это просто JSON.

> Источник правды по тому, КТО что сделал — маркетплейс-репо `skills-marketplace` (`@<handle>/verified/skills`, `verified/scenarios`, `artifacts.md`, `scenario.md`). `data.json` — это его витрина: после того как разложил у сотрудника скилл/сценарий/артефакт по папкам, отрази это здесь.

## Структура файла

```jsonc
{
  "scen":  [ … ],   // доска №1 «Респект · сценарии»  (за сценарии — только респект, без денег)
  "money": [ … ],   // доска №2 «Отдано команде · скиллы/артефакты» (+ $100 за командную ценность)
  "cat":   [ … ]    // каталог «Что есть и зачем» (карточки с фильтром Сценарии/Скиллы/Артефакты)
}
```

### `scen` — респект-доска (сценарии)
Один объект на сотрудника. Тир (💎/🥇/🥈/🥉) и сортировка считаются автоматически по `c`.
```jsonc
{ "n": "Рома", "h": "@classDev", "role": "СТО", "c": 4 }
```
- `n` — имя в доске · `h` — telegram-хэндл (с `@`) · `role` — роль · `c` — **число сценариев** у человека.
- `c: 0` → в доске покажется «—» и без тира.

### `money` — доска «отдано команде» (скиллы/артефакты)
```jsonc
{ "n": "Рома", "h": "@classDev", "role": "СТО", "s": 1, "a": 1, "money": 0 }
```
- `s` — сколько **скиллов** отдал · `a` — сколько **артефактов** · сумма `s+a` идёт в столбец «Отдал» и в тир.
- `money` — выплачено **$** (0 → в столбце «$» покажется «впереди»). Ставить >0, когда скилл/артефакт реально набрал ≥10 командных вызовов за месяц и премия согласована.

### `cat` — карточки каталога
```jsonc
{
  "type": "scenario",                 // "scenario" | "skill" | "artifact" — определяет чип и вкладку
  "name": "night-cap-handler",        // заголовок карточки (для скилла = имя в репо)
  "cmd": "/top-offers",               // НЕОБЯЗ.: команда вызова — показывается строкой «команда: …»
  "st": "в маркетплейсе",             // статус-подпись справа вверху (любой текст)
  "dn": "Рома",                       // имя девелопера/автора (футер карточки)
  "dh": "@classDev",                  // его хэндл
  "link": "https://…",                // НЕОБЯЗ.: если есть — name становится кликабельным ↗
  "desc": "Что делает — 1–2 предложения.",
  "val":  "Выгода команде — 1 предложение."
}
```

Правила контента (полностью — в `REQUIREMENTS.md` рядом):
- **Артефакт со ссылкой** → обязательно `link` (карточка кликабельна).
- **Скилл** → `cmd` (команда вызова) + `dh` (под чьим хэндлом в репо).
- **Шаблонные однотипные скиллы/сценарии** (как Викины 26 сверок) → **одна обобщённая карточка**, а количество прогонов — в `st` и в счётчике доски, не плодить 26 карточек.
- `type` карточки = где сущность лежит в маркетплейсе: `verified/skills` → `skill`; `scenario.md` / `verified/scenarios` → `scenario`; `artifacts.md` → `artifact`.

## Как выкатить

```bash
# 1) поправил data.json, проверил что это валидный JSON:
python3 -m json.tool data.json >/dev/null && echo OK

# 2) запушил в репо:
git add data.json && git commit -m "data: <что поменял>" && git push

# 3) на сервере подтянул и пересобрал контейнер:
sudo git -C /root/tsl-rewards pull --ff-only
sudo sh -c "cd /root/tsl-rewards && docker compose up -d --build"
```

Проверка: открой https://tsl-skills.com (под своей телегой) — изменения видны после обновления страницы.
Если `data.json` сломан/недоступен — сайт не падает, а показывает последний вшитый снапшот (деградация). Поэтому всегда прогоняй шаг 1.

## Чего НЕ трогать
- `auth_app.py`, `Dockerfile`, `docker-compose.yml`, `nginx.conf`, `assets/`, `index.html` — это код/гейт/сборка фронта (моя зона, Алексея). Данные — только `data.json`.
- Список допущенных телег (кто вообще может зайти на сайт) — в `/root/tsl-rewards/.env` (`ALLOWED_USERNAMES`), не в `data.json`.
