# Лидерборд tsl-skills.com — как менять данные

Все данные сайта (две доски + карточки каталога) лежат в **одном файле на сервере**:

```
/root/tsl-rewards-data/data.json      ← правишь ТЫ (нужен sudo, файл root-owned)
```

Контейнер сайта примонтировал этот файл и читает его на **каждый запрос**. Поэтому:
**поправил файл → обновил страницу в браузере → всё, изменения уже live.**
Ни git, ни push, ни пересборка контейнера, ни рестарт — **не нужны**.

> Источник правды по тому, КТО что сделал — маркетплейс-репо `skills-marketplace`
> (`@<handle>/verified/skills`, `verified/scenarios`, `artifacts.md`, `scenario.md`).
> `data.json` — его витрина: разложил у сотрудника скилл/сценарий/артефакт по папкам → отрази это здесь.

## Структура файла

```jsonc
{
  "scen":  [ … ],   // доска №1 «Респект · сценарии»  (за сценарии — только респект, без денег)
  "money": [ … ],   // доска №2 «Отдано команде · скиллы/артефакты» (+ $100 за командную ценность)
  "cat":   [ … ]    // каталог «Что есть и зачем» (карточки с фильтром Сценарии/Скиллы/Артефакты)
}
```

### `scen` — респект-доска (сценарии)
Один объект на сотрудника. Тир (💎/🥇/🥈/🥉) и сортировка считаются автоматически по `c`.
```jsonc
{ "n": "Рома", "h": "@classDev", "role": "СТО", "c": 4 }
```
- `n` — имя · `h` — telegram-хэндл (с `@`) · `role` — роль · `c` — **число сценариев**. `c: 0` → «—» без тира.

### `money` — доска «отдано команде» (скиллы/артефакты)
```jsonc
{ "n": "Рома", "h": "@classDev", "role": "СТО", "s": 1, "a": 1, "money": 0 }
```
- `s` — скиллов отдал · `a` — артефактов · сумма `s+a` → столбец «Отдал» и тир.
- `money` — выплачено **$** (0 → «впереди»). Ставить >0, когда скилл/артефакт набрал ≥10 командных вызовов за месяц и премия согласована.

### `cat` — карточки каталога
```jsonc
{
  "type": "scenario",                 // "scenario" | "skill" | "artifact" — чип и вкладка
  "name": "night-cap-handler",        // заголовок (для скилла = имя в репо)
  "cmd": "/top-offers",               // НЕОБЯЗ.: команда вызова → строка «команда: …»
  "st": "в маркетплейсе",             // статус-подпись справа вверху (любой текст)
  "dn": "Рома", "dh": "@classDev",    // девелопер/автор + его хэндл (футер карточки)
  "link": "https://…",                // НЕОБЯЗ.: если есть — name кликабельный ↗
  "desc": "Что делает — 1–2 предложения.",
  "val":  "Выгода команде — 1 предложение."
}
```
Правила контента (полностью — в `REQUIREMENTS.md` в репо `cj-sketch/tsl-rewards`):
- **Артефакт со ссылкой** → обязательно `link`. **Скилл** → `cmd` + `dh`.
- **Шаблонные однотипные** (как 26 Викиных сверок) → **одна обобщённая карточка**, количество — в `st`/счётчике, не плодить N карточек.
- `type` = где сущность в маркетплейсе: `verified/skills`→`skill`; `scenario.md`/`verified/scenarios`→`scenario`; `artifacts.md`→`artifact`.

## Как поменять (3 шага, без сборки)

```bash
# 1) правишь файл (ты root — через sudo; или твой агент):
sudo nano /root/tsl-rewards-data/data.json

# 2) ОБЯЗАТЕЛЬНО проверь, что JSON не сломан:
sudo python3 -m json.tool /root/tsl-rewards-data/data.json >/dev/null && echo OK

# 3) открой https://tsl-skills.com под своей телегой — изменения уже видны (обнови страницу).
```
Контейнер трогать не надо — он читает файл на каждый запрос. Сломаешь JSON — сайт не упадёт, покажет прошлый снапшот (деградация), поэтому шаг 2 обязателен.

## Чего НЕ трогать
- `/root/tsl-rewards/` — это сам проект (гейт `auth_app.py`, `Dockerfile`, `assets/`, `index.html`, compose). Код/гейт/сборка — зона Алексея.
- Список допущенных телег (кто вообще заходит на сайт) — в `/root/tsl-rewards/.env` (`ALLOWED_USERNAMES`).
- Правишь **только** `/root/tsl-rewards-data/data.json`.
