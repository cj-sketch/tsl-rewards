# TSL Rewards — требования к контенту сайта

Принципы, **как сайт показывает сущности**. Читать перед правкой каталога/досок.
Tech-agnostic: правила про контент, не про фреймворк (Tailwind/HeroUI — деталь реализации).

## Сущности и что про них показываем

### Сценарий
Личная автоматизация. На сайте — **только имя + место/тир на респект-доске**. Описания нет (сценарии не для переиспользования).

### Скилл
Переиспользуемая `/команда` из маркетплейса. В каталоге показываем:
- **команда вызова** — как скилл зовётся (напр. `/top-offers`);
- **под чьим хэндлом заведён в репо** — `<handle>/verified/skills/<name>` (это же владелец/девелопер);
- что делает + выгода команде;
- статус (в маркетплейсе / adoption ×N / начислено $).
- **Шаблонные скиллы** (одно и то же в разных контекстах — как сверки Вики по разным адвертам): **НЕ плодить карточку на каждый**. Давать **одну обобщённую** карточку: суть + количество + пример перечня (напр. «Сверки адвертов · 26 шт» со списком адвертов и обобщённой командой вызова).

### Артефакт
Самостоятельный продукт (дашборд / бот / сервис). В каталоге:
- **если есть ссылка — название кликабельно** (ведёт на продукт/бота/ЛК);
- что делает + выгода + девелопер + статус.

## Источник данных
Каталог и счётчики берутся из инвентаризации маркетплейса `team-skills-marketplace/`
(`<handle>/verified/{skills,scenarios,artifacts}`). Тексты «выгода» финализирует девелопер.

## Технический стек (на 2026-06-15)
Основной вариант — **React 19 + Vite + HeroUI v3** (исходник в `Business/rewards-site-heroui/`,
билд-вывод деплоится в репо `cj-sketch/tsl-rewards`). Предыдущий лёгкий Tailwind-вариант
(`Business/rewards-site/index.html`) сохранён как референс.

## Разделы сайта (порядок, обязательные)
hero → принцип ($100 за командную ценность) → **01 Лидерборды** (две доски рядом) → **02 Каталог** (скиллы/артефакты) → **03 Премии** (3 типа + путь к $100) → **04 База про агентов** — видео Loom (`loom.com/embed/8b05bf9e10cc4ca1b0f103a580dc38db`), Алексей ~15 мин про агентов/сценарии/скиллы. **Не выкидывать при пересборках.**
